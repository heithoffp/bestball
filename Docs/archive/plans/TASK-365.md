# TASK-365: Data-load performance overhaul: pipeline optimization + web compact ADP artifact + web entries cache

**Status:** Approved
**Priority:** P2

---

## Objective

Cut the 4-5 second data load on every web reload and iOS launch to well under one second by
fixing the three measured bottlenecks: quadratic scans in the shared `processMasterList`
pipeline (~3.8s, both platforms), per-reload PapaParse of 134 raw ADP CSVs on web (~1.7s,
13 MB), and the web's uncached full refetch of `extension_entries` from Supabase on every
reload.

## Verification Criteria

1. **Reloading the signed-in web app paints the portfolio in well under a second** (was 4-5s),
   and relaunching the iOS app does the same — with all tab values (Exposures percentages,
   ADP Tracker timelines, Dashboard counts) identical to before the change.
2. **A parity check proves the optimized pipeline is output-identical**: old vs. new
   `processMasterList` produce deep-equal results on the real 134-snapshot ADP bundle plus a
   synthetic multi-entry portfolio, with the new version at least 10× faster.
3. **The web bundle no longer ships raw ADP CSVs** — the 13 MB of `src/assets/adp/*.csv` is
   replaced in the build output by one ~2 MB compacted JSON artifact, and new ADP snapshot
   CSVs still flow to the site with the existing "drop CSV, deploy" workflow (the artifact
   regenerates automatically on every build/dev run).

## Verification Approach

Automated (Claude runs and reports):

- **Pipeline parity + speed benchmark**: a scratchpad Node script loads
  `mobile-app/shared/data/adpSnapshots.json`, builds a synthetic 150-entry portfolio, runs the
  pre-change `processMasterList` (from git) and the optimized one, asserts
  `deepStrictEqual` of outputs, and prints both timings. Pass = identical output, new time
  ≤ 10% of old (baseline measured this session: 3.8s → 0.27s).
- **Web**: `npm run lint` and `npm run build` in `best-ball-manager/` exit 0 (the project has
  no test suite — lint + build is the verification path per TASK-362).
- **Bundle check**: after `npm run build`, `dist/assets/` contains no per-CSV ADP chunks and
  total dist size drops by roughly 10 MB.
- **Artifact freshness**: `predev`/`prebuild` hooks regenerate
  `src/data/adpSnapshots.json`; verify by touching a new dated CSV into `src/assets/adp/`,
  running the build, confirming the artifact picks it up, then removing the test CSV.
- **Mobile**: `mobile-app/shared/utils/helpers.js` changes verified via babel-preset-expo
  transform (mobile has no lint script). Re-run `npm run build:data` and confirm it still
  succeeds.
- **Entries cache logic**: `mergeEntries` and the cache read/write round-trip exercised in a
  Node script against fixture data (upsert, edit, delete reconciliation, corrupt-payload
  miss, user-mismatch miss).

Requires the developer (manual):

1. Web: sign in on the dev server, hard-reload twice; second reload should paint the
   portfolio near-instantly, then quietly reconcile. Spot-check Exposures / ADP Tracker /
   Dashboard against pre-change values.
2. Web: delete a roster, reload — it must stay gone (cache reconciliation).
3. iOS: launch-time improvement rides the next EAS build (run `npm run build:data` first, per
   the standard build workflow); verify at that point, not as part of this task's close.

## Implementation Approach

**Step 1 — Pipeline optimization (shared, biggest win: ~3.5s both platforms).**
In both `helpers.js` copies:

- Add a module-level `Map` memo inside `canonicalName` (keyed by the raw input string;
  bounded by unique player-name strings, a few thousand entries).
- In `processMasterList`:
  - Build `rosterByName: Map<normName, rosterRow>` (first occurrence) in the same pass that
    builds `draftCounts` — replaces the per-universe-player `rosters.find(...)` at
    `helpers.js:166`.
  - Build `adpByNorm: Map<normName, adpObj>` once from `Object.keys(adpMap)` — replaces the
    two `Object.keys(adpMap).find(k => normalize(k) === normName)` scans at lines 182/196.
  - Everything else (snapshot lookups, history arrays, display-name precedence, sort order,
    all field-fallback chains) stays behavior-identical.
- Keep the two files in lockstep per the CLAUDE.md port-snapshot rule.

**Step 2 — Web compact ADP artifact (~1.7s + 11 MB, web only).**

- Adapt `mobile-app/scripts/build-data.mjs` into `best-ball-manager/scripts/build-data.mjs`,
  emitting only the ADP artifact (projections/rankings/demo rosters stay as-is on web —
  they're small). Platform/date filename parsing mirrors `loadBundledAdp()` including the
  superflex/eliminator prefixes and the `1900-01-01` latest-fallback guard.
- Wire `predev` + `prebuild` npm hooks so the artifact can never go stale relative to
  `src/assets/adp/*.csv` — the existing "drop a CSV, redeploy" workflow is unchanged, and
  Vercel builds regenerate it automatically.
- `bundledAdp.js` statically imports the JSON and decodes with the mobile `decodeAdp` logic;
  `App.jsx` call sites swap `loadBundledAdp()` for it. `dataLoader.js` uses
  `snap.rows ?? await parseCSVText(text)` so both shapes work during the transition.
- `src/assets/adp/` stays read-only and untouched — it remains the source of truth the
  script reads from.

**Step 3 — Web entries cache (removes the Supabase round-trip from the critical path).**

- Port `entriesCache.js` verbatim except the storage layer: payload
  `{version, userId, cursor, entries}` stored as JSON text through the existing IndexedDB
  helpers (id `entries_cache`). Corrupt/mismatched payloads are treated as a miss and
  deleted, exactly like mobile.
- `App.jsx` `loadFromExtension()` adopts the mobile `PortfolioContext` flow: cache hit →
  render immediately → background `refreshEntries` (delta by `synced_at` cursor + live-id
  deletion reconciliation) → re-apply only if changed. Cache miss → full fetch, write cache.
- `handleDeleteRoster` also rewrites the cache (as mobile's `deleteRoster` does) so a reload
  doesn't resurrect the row. Sign-out is already covered: the payload's `userId` check makes
  a stale cache a miss, and `clearAllData()` clears the store.

**Sequencing:** Step 1 first (shared, benchmark-proven), then 2, then 3 — each independently
shippable and verifiable.

**ADR note:** No new ADR proposed — Steps 2 and 3 extend decisions already recorded for
mobile (ADR-030 entries cache, ADR-031 ADP artifact) to the web platform, and Step 1 is a
behavior-preserving optimization. A short ADR documenting "web adopts the mobile caching
architecture" can be recorded instead if preferred.

## Scope Addendum (during implementation)

- **`mobile-app/scripts/build-data.mjs` — one-line `roundAdp` fix (2026-07-23).** The web
  CSV-vs-artifact parity drill exposed an inherited bug: `Number('') === 0`, so a player
  absent from a snapshot got ADP `0` instead of `null` (53k history entries affected;
  mobile ships this today). Fixed in both scripts' `roundAdp` and both artifacts
  regenerated. Verification: parity drill reports 0 beyond-rounding ADP mismatches.
- **Known, accepted differences vs the raw-CSV path** (both match what mobile already
  ships): ADP values round to 2 decimals (max delta 0.005, UI shows 1 decimal);
  `projectedPoints` for 118 deep-bench players absent from both the latest platform
  snapshots and `projections.csv` (e.g. Philip Rivers, Zach Wilson) now come from the
  positional-neighbor backfill instead of stale historical snapshots.
